# Camera Wiring - ESP32-S3

## Overview

This document consolidates wiring information for OV2640, OV7670, and OV7675 camera modules on the LilyGO T-SIM7080G-S3 board. All three cameras use the DVP (Digital Video Port) interface with identical GPIO pin assignments.

## Hardware Platform

**Target Board**: LilyGO T-SIM7080G-S3
- **Microcontroller**: ESP32-S3-WROOM-1-N16R8
- **Camera Interface**: DVP (Digital Video Port) - 8-bit parallel interface
- **Connection Methods**:
  - OV2640: Flat cable connector on board
  - OV7670/OV7675: Header pins (jumper wires)

## Camera Interface: DVP (Digital Video Port)

DVP is an 8-bit parallel interface used by all three camera modules. It consists of:

### Control Interface (I2C/SCCB)
- **SIOC** (I2C Clock): GPIO01
- **SIOD** (I2C Data): GPIO02
- **Purpose**: Camera configuration and detection

### Video Data Interface
- **XCLK** (External Clock): GPIO08 - Clock signal generated by ESP32-S3
- **VSYNC** (Vertical Sync): GPIO16 - Frame synchronization
- **HREF** (Horizontal Reference): GPIO17 - Line synchronization
- **PCLK** (Pixel Clock): GPIO12 - Pixel data clock
- **Data Bus (8-bit)**: GPIO14, 47, 48, 21, 13, 11, 10, 09 (D0-D7)
- **Reset**: GPIO18 (optional but recommended)
- **PWDN**: Not used (-1)

## GPIO Pin Mapping for T-SIM7080G-S3

All cameras use the same GPIO pin assignments:

## GPIO Conflicts

### Critical Conflict
- **GPIO39** (SDMMC CMD) conflicts with camera D5
- **Solution**: Unmount SD card before initializing camera

### Other Potential Conflicts
- **GPIO12**: Shared between FSPICLK (SPI) and PCLK (Camera)
- **GPIO11**: Shared between FSPID (SPI) and D5 (Camera)
- **GPIO13**: Shared between FSPIQ (SPI) and D4 (Camera)
- **GPIO10**: Shared between FSPICS0 (SPI) and D6 (Camera)

Ensure SD card is unmounted and SPI is not in use when camera is active.

## OV2640 Wiring Guide

**Connection Method**: Flat cable connector on T-SIM7080G-S3 board

### Pin Mapping (via Flat Cable Connector)

The OV2640 connects via a flat flexible cable (FFC) to the dedicated connector on the board. The pin mapping is:

| Signal | GPIO | Description |
|--------|------|-------------|
| XCLK | GPIO08 | External clock |
| I2C SCL | GPIO01 | I2C clock |
| I2C SDA | GPIO02 | I2C data |
| VSYNC | GPIO16 | Vertical sync |
| HREF | GPIO17 | Horizontal reference |
| PCLK | GPIO12 | Pixel clock |
| D0-D7 | GPIO14,47,48,21,13,11,10,09 | Data bus |
| RESET | GPIO18 | Reset (optional) |

### Hardware Setup Instructions

1. **Power off board**: Unplug USB-C connector
2. **Inspect flat cable**: Ensure no bent pins or damage
3. **Seat flat cable**: Insert FFC into connector on board, ensuring proper alignment
4. **Verify connection**: Flat cable should be fully seated with no gaps
5. **Check power**: Verify 3.3V power supply is stable (provided by board)

### Critical Connections

- **I2C lines** (GPIO01/GPIO02): Must be correct for camera detection
- **XCLK** (GPIO08): Required for frame generation
- **All DVP data pins** (D0-D7): All must be connected for image data transfer

## OV7670 Wiring Guide

**Connection Method**: Header pins (jumper wires)

### OV7670 Pinout

OV7670 modules typically have a **double-row header** with the following pin arrangement:

**Left Column (Inner Row):**
| Position | Label | Signal | Description |
|----------|-------|--------|-------------|
| 1 | 3.3V | VCC | 3.3V Power |
| 2 | SCL | SIO_C | I2C Clock (SCCB) |
| 3 | VS | VSYNC | Vertical Sync |
| 4 | PLK | PCLK | Pixel Clock |
| 5 | D7 | D7 | Data Bit 7 (MSB) |
| 6 | D5 | D5 | Data Bit 5 |
| 7 | D3 | D3 | Data Bit 3 |
| 8 | D1 | D1 | Data Bit 1 |
| 9 | RET | RESET | Reset |

**Right Column (Outer Row):**
| Position | Label | Signal | Description |
|----------|-------|--------|-------------|
| 1 | DGND | GND | Ground |
| 2 | SDA | SIO_D | I2C Data (SCCB) |
| 3 | HS | HREF | Horizontal Reference |
| 4 | XLK | XCLK | External Clock Input |
| 5 | D6 | D6 | Data Bit 6 |
| 6 | D4 | D4 | Data Bit 4 |
| 7 | D2 | D2 | Data Bit 2 |
| 8 | D0 | D0 | Data Bit 0 (LSB) |
| 9 | PWDN | PWDN | Power Down (not used) |

**Note**: This is a double-row header. The left column (inner row) and right column (outer row) are on the same physical header connector.

### Step-by-Step Wiring Instructions

1. **Power Off**: Unplug USB-C connector from board
2. **Connect Power**:
   - Connect OV7670 **VCC** to **P1.1 (3.3V)** or **P2.1 (3.3V)** on LilyGO
   - Connect OV7670 **GND** to **P1.2 (GND)** or **P2.15 (GND)** on LilyGO
3. **Connect I2C Control** (CRITICAL):
   - Connect OV7670 **SCL** (SIO_C) to **P2.3 (GPIO01)** on LilyGO
   - Connect OV7670 **SDA** (SIO_D) to **P2.4 (GPIO02)** on LilyGO
4. **Connect Clock**:
   - Connect OV7670 **XCLK** to **P1.6 (GPIO08)** on LilyGO
5. **Connect Control Signals**:
   - Connect OV7670 **VSYNC** to **P1.3 (GPIO16)** on LilyGO
   - Connect OV7670 **HREF** to **P1.4 (GPIO17)** on LilyGO
   - Connect OV7670 **PCLK** to **P1.12 (GPIO12)** on LilyGO
6. **Connect Reset** (optional but recommended):
   - Connect OV7670 **RESET** to **P1.5 (GPIO18)** on LilyGO
7. **Connect Data Bus (D0-D7)**:
   - Connect OV7670 **D0** to **P1.14 (GPIO14)** on LilyGO
   - Connect OV7670 **D1** to **P2.13 (GPIO47)** on LilyGO
   - Connect OV7670 **D2** to **P2.12 (GPIO48)** on LilyGO
   - Connect OV7670 **D3** to **P2.14 (GPIO21)** on LilyGO
   - Connect OV7670 **D4** to **P1.13 (GPIO13)** on LilyGO
   - Connect OV7670 **D5** to **P1.11 (GPIO11)** on LilyGO
   - Connect OV7670 **D6** to **P1.10 (GPIO10)** on LilyGO
   - Connect OV7670 **D7** to **P1.9 (GPIO09)** on LilyGO
8. **Double-Check**: Review all connections carefully. Ensure no wires are loose or incorrectly placed.

### Critical I2C Connections

The I2C connections are essential for camera detection:
- **SCL** (left column, position 2) → **GPIO01** (P2.3)
- **SDA** (right column, position 2) → **GPIO02** (P2.4)

### Differences from OV7670

OV7675 and OV7670 use **identical wiring** and pin configurations. The only differences are:
- **Sensor PID**: OV7675 has PID=0x75, OV7670 has PID=0x76
- Both use the same GPIO pin assignments

## Wiring Best Practices

### Wire Length and Quality

- **Keep wires short**: Jumper wires should be as short as possible (ideally < 10 cm)
- **Longer wires**: Can cause signal integrity issues and frame capture failures
- **Twisted pairs**: Use twisted pairs for data lines if wires are longer than 5 cm
- **Wire quality**: Use good quality jumper wires (22-24 AWG recommended)

### Power Supply

- **Voltage**: All cameras require 3.3V power supply
- **Current capacity**: Ensure adequate current capacity (cameras typically draw 20-40 mA)
- **Stability**: Verify stable 3.3V power supply
- **Decoupling**: Use decoupling capacitors if power supply is noisy

### Signal Integrity

- **Wire routing**: Avoid crossing power and signal wires
- **Data bus**: Keep data bus wires parallel and close together
- **Connection stability**: Use breadboard or protoboard for stable connections
- **Loose connections**: Ensure all wires are firmly seated

### I2C Pull-up Resistors

- **Internal pull-ups**: ESP32-S3 has internal pull-up resistors
- **External pull-ups**: May be needed for some modules or longer wires
- **Resistor value**: If needed, add external 4.7kΩ pull-up resistors from GPIO01 (SCL) to 3.3V and from GPIO02 (SDA) to 3.3V

### Pin Verification

- **Module pinout**: Always verify your specific camera module's pinout using its datasheet or silk screen labels
- **Pin numbers**: Pin numbers may vary by module manufacturer
- **Double-check**: Review all connections before powering on

## Sensor Detection Reference

After wiring, the ESP-IDF camera driver will detect the sensor via I2C. The sensor PID identifies the camera type:

| Camera | Sensor PID | Detection Message |
|--------|------------|-------------------|
| OV2640 | 0x26 | `Detected sensor: PID=0x26` |
| OV7675 | 0x75 | `Detected sensor: PID=0x75` |
| OV7670 | 0x76 | `Detected sensor: PID=0x76` |

## Connection Summary

### Total Connections Required

**OV2640** (via flat cable):
- 2 Power (VCC, GND) - via FFC
- 2 I2C (SIOC, SIOD) - via FFC
- 4 Control (VSYNC, HREF, PCLK, XCLK) - via FFC
- 8 Data (D0-D7) - via FFC
- 1 Reset (optional) - via FFC

**OV7670/OV7675** (via header pins):
- 2 Power (VCC, GND)
- 2 I2C (SIO_C, SIO_D)
- 4 Control (VSYNC, HREF, PCLK, XCLK)
- 8 Data (D0-D7)
- 1 Reset (optional but recommended)
- 1 PWDN (not used)

### Common Pin Assignments

All three cameras use the same GPIO pin assignments, making the code configuration identical for all camera types. The only difference is the connection method (flat cable for OV2640 vs header pins for OV7670/OV7675).
