## Implemented run flow (verified)

1. A single experiment run starts in `scripts/experiment_set_match/run_experiment.py`.
2. The script starts `scripts/image_slider_web/server.py` as a subprocess, optionally opens the browser, and waits for a new slider `run_log.csv` folder under `scripts/image_slider_web/outputs/`.
3. The slider frontend starts a run via `/start`, loads `/manifest.json`, shows a 20 second calibration screen, then shows one image every 2 seconds, and POSTs `shown_at_iso` rows to `/log`.
4. In parallel, experiment code reads serial bytes, extracts JSON objects, keeps only strict-valid `name == "INVOKE"` payloads, and logs parse errors for malformed objects/events.
5. After 388 frames (default), events are assigned to frame windows `[t0, t0+2.0)` using host timestamps.
6. Per frame, GT class sets come from `data/test/labels/*.txt` (with filename token checks and NUL conflict checks). Predicted class sets are derived from detections with `conf >= 0.30`.
7. Outputs are written to `outputs/<yyyy-mm-dd_HHMMSS>__<model_name>/` with per-image results, review candidates, metrics, raw/unmatched events, parse errors, and run config.

## What is automated vs manual for the "5 model" test

### Automated now

- End-to-end data capture and scoring for one model run once the script is started.
- Creation of isolated output folder per run with configuration and diagnostics.
- Presence-based per-class TP/FP/FN and overall exact set match rate computation.

### Not automated now

- Multi-model loop/orchestration: there is no script that iterates through 5 models and produces a combined comparison table.
- Model flashing/swap between runs remains manual ([to be verified] exact operator procedure is outside current code tree).
- Run start may still be operator-dependent (browser interaction and optional pause/next/restart controls can alter timing behavior).

## Metrics and artifacts produced per run (verified)

- `per_image_results.csv`: frame-level GT/pred sets, timeout, match type, raw detections, sampling diagnostics.
- `review_candidates.csv`: subset for mismatch, multi-species, or timeout review.
- `per_class_metrics.csv`: TP/FP/FN, support, precision, recall (precision/recall become `NA` if denominator is zero).
- `raw_invoke_events.jsonl` and `unmatched_invoke_events.jsonl`.
- `parse_errors.json`: parse error counts/details.
- `run_config.json`: threshold, windows, grace, sampling offsets, serial settings, slider metadata.

## Implementation status update (2026-03-28)

1. **5-model automation wrapper: implemented**
   - Added `scripts/experiment_set_match/run_all_models.py`.
   - Supports fixed ordered `--model-name` (exactly five), operator checkpoint per model, and session-level `aggregate_summary.csv`.

2. **Sampling-window policy alignment: implemented**
   - Policy standardized on `+-200 ms` and documented accordingly.
   - `SAMPLING_MATCH_WINDOW_S` remains `0.200` in `run_experiment.py`.

3. **Slider README command inconsistency: implemented**
   - README run commands standardized to `--port` syntax.
   - Locked benchmark startup path documented.

4. **Operator control drift risk: implemented**
   - Added strict locked benchmark mode in slider server (`--locked-benchmark`) and frontend enforcement.
   - Added `POST /log/control` and `control_events.jsonl` for audit logging.

5. **Serial port auto-selection risk: implemented**
   - Added explicit `--serial-port`.
   - Glob fallback now fails fast on ambiguous matches.
   - `run_config.json` now includes `serial_port_resolved` and selection mode.

6. **Reproducibility linkage policy: implemented**
   - Per-run metadata sidecars now include `run_meta.json` (slider) and enriched `run_config.json` (experiment).
   - Cross-run machine-readable linkage is appended to `scripts/image_slider_web/runs_ledger.jsonl`.
   - Session aggregation is written to `outputs/<timestamp>__five_model_session/aggregate_summary.csv`.
